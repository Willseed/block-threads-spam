# 目的

候選偵測的目標是縮小人工審核範圍，而不是自動判定詐騙。系統只針對已確認正式帳號建立可解釋且有上限的候選集合，並允許使用者人工加入已知疑似帳號。

# 候選來源

| 來源 | 說明 | 控制 |
|---|---|---|
| 視覺混淆替換 | 常見字母與數字混淆，例如字母 o 與數字 0、字母 l 與數字 1 | 僅使用平台允許且經審核的替換表 |
| 標點變形 | 句點與底線的加入、移除或位置變化 | 只在原名稱分段附近產生 |
| 單字元編輯 | 一次刪除、重複、插入或相鄰交換 | 限制編輯距離與候選總數 |
| 前後綴 | 加入少量常見冒用前後綴 | 使用受控詞表，不接受任意大量字典 |
| 人工目標 | 使用者或受信任案件來源提供的完整帳號名稱 | 格式驗證、去重與租戶範圍限制 |
| 歷史監看 | 先前存在但尚未處置的候選 | 以較低頻率重新檢查 |

系統不應對所有可能字元做笛卡兒積排列，也不應將平台搜尋結果無限制擴張成新的掃描來源。

# 變形範圍策略

每次掃描先建立候選快照，套用：

* 每個正式帳號的候選總上限。
* 每種變形規則的配額。
* 每日新候選上限。
* 已檢查不存在帳號的退避週期。
* 已忽略候選的低頻或停止檢查設定。
* 使用者可檢視的「候選如何產生」說明。

# 相似度訊號

| 訊號 | 用途 | 限制 |
|---|---|---|
| 使用者名稱差異 | 衡量視覺與編輯距離 | 不能單獨作為冒用結論 |
| 顯示名稱相似 | 發現同名或近似名稱 | 常見姓名可能造成高誤判 |
| 頭像相似 | 發現重用或輕微修改的照片 | 只保存必要衍生指紋與證據；結果供人工判斷 |
| 簡介文字重疊 | 發現複製介紹、職稱或品牌敘述 | 避免擷取與案件無關的大量內容 |
| 外部連結相似 | 發現相同或近似網站、聯絡方式 | 連結存在不等於惡意，需使用者核對 |
| 帳號互動跡象 | 顯示是否提及或接觸正式帳號 | 僅在使用者工作階段可合法看到且確有必要時使用 |

# 優先級而非判決

系統輸出「低、中、高審核優先級」，並列出造成分數的理由。例如「使用者名稱只有一個視覺混淆字元」或「頭像與顯示名稱皆高度相似」。介面避免使用「已確認詐騙」等定論字樣，除非使用者自行標記並承擔該判斷。

# 候選狀態

1. 新發現：尚未由使用者查看。
2. 待審核：已存在且有證據。
3. 持續監看：使用者尚不處置，保留低頻複查。
4. 已忽略：使用者判定不需處置。
5. 準備封鎖：使用者進入封鎖確認，但尚未批准。
6. 封鎖執行中：單一目標工作已取得鎖。
7. 已封鎖：結果明確確認。
8. 待人工複查：操作結果不明或平台介面改變。
9. 不存在：目前無法找到，依退避策略重新檢查。

# 證據最小化

每個候選保存一張以個人檔案為主的證據與結構化相似理由。除非案件需要，不保存完整動態消息、私人訊息、追蹤者清單或無關使用者資料。
