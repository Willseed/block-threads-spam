# 全域導覽

網頁提供「儀表板、候選帳號、活動紀錄、連線與設定」五個主要區域。頁首固定顯示目前選取的正式 Threads 帳號、工作階段健康狀態與最後掃描時間。

# 畫面一：應用程式登入

主要內容：登入方式、服務用途、隱私摘要、帳號救援與安全說明。

安全要求：支援強式驗證；高風險流量使用 Turnstile 與速率限制；登入成功後使用安全 Cookie，不把敏感權杖保存在前端可讀儲存。

# 畫面二：連線 Threads

主要內容：為何需要 Threads OAuth、會要求哪些權限、token 如何加密保存、如何撤銷、開始連線按鈕。

連線開始後導向 Meta 官方 OAuth 頁面。返回本服務後固定顯示：

* 請只在 Meta／Threads 官方頁面輸入帳密。
* 本服務不會要求你貼上 Cookie 或 Session 檔。
* 授權返回後仍需確認完整 Threads username。
* 隨時可以取消；未完成或不相符的連線不會啟用掃描。

# 畫面三：確認正式帳號

主要內容：OAuth token 交換後由官方身分端點讀取到的完整使用者名稱、顯示名稱、頭像與個人檔案摘要。

主要動作：「確認並保護此帳號」與「不是這個帳號，重新登入」。確認後才建立候選規則。

# 畫面四：儀表板

主要區塊：

* 連線健康：有效、需要重新登入、暫停或已撤銷。
* 候選摘要：新發現、待審核、高優先、監看中、已封鎖。
* 最新證據：最近出現的高優先候選。
* 掃描控制：立即掃描、暫停排程、查看下一次排程。
* 安全通知：平台挑戰、工作階段失效或異常動作。

# 畫面五：候選清單

每張候選卡片顯示：

* 完整 Threads 使用者名稱。
* 與正式帳號的差異標示。
* 優先級與可解釋理由。
* 顯示名稱、頭像縮圖與最近檢查時間。
* 狀態與可用動作。

篩選器支援狀態、優先級、變形類型與日期；不提供可對大量候選直接封鎖的全選按鈕。

# 畫面六：候選詳情

主要內容：正式帳號與候選並排比較、完整證據、名稱差異、頭像與簡介相似訊號、歷史檢查、使用者決策與活動紀錄。

主要動作：「忽略」、「持續監看」、「封鎖此帳號」。封鎖按鈕進入獨立確認頁，不在詳情頁立即執行。

# 畫面七：封鎖確認

固定顯示：

* 將使用的正式 Threads 帳號。
* 將被封鎖的完整目標名稱。
* 最新重新載入時間與證據。
* 封鎖只影響該 Threads 帳號與目標之間的關係。
* 這不是向 Meta 提交冒用檢舉。

使用者完成再驗證並給予單一目標批准。執行期間顯示不可重複提交的工作狀態。

# 畫面八：工作結果

結果只使用三種清楚狀態：

* 已確認成功。
* 已停止，未執行。
* 結果不明，需要人工複查。

畫面提供封鎖前後證據、工作時間、目標、錯誤分類與下一步。結果不明時不顯示「再試一次」捷徑，避免立即重複破壞性動作。

# 畫面九：活動紀錄

顯示登入連線、工作階段更新、掃描、候選決策、封鎖批准、執行結果、證據存取與撤銷。使用者可依 Threads 帳號與日期篩選。

# 畫面十：連線與設定

主要內容：已連線 Threads 帳號、工作階段健康、候選規則、掃描頻率、通知、證據保留與刪除、中斷連線。

中斷連線需要再驗證，並清楚說明會停止排程、刪除工作階段及如何處理歷史證據。

設定頁另需說明 Meta 解除授權或 data deletion 只處理相符 Threads 連線資料，不等同刪除 Cloudflare Access 帳號或整個本服務租戶。目前完整「刪除本服務帳號」控制仍待實作，不可顯示為已完成能力。
