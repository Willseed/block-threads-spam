# 服務承諾

本服務協助帳號本人更快發現可能模仿其使用者名稱的 Threads 個人檔案，集中檢視證據，並用自己的已登入工作階段完成封鎖。它不是身分認證機關，也不替 Meta 判定某個帳號是否構成冒用或詐騙。

# 必要條件

* 使用者必須先通過本服務的身分驗證。
* Threads OAuth 連線必須由目前登入本服務的同一位使用者主動開始，state 必須綁定其 Access session 並只能消耗一次。
* 使用者只在 Meta 官方頁面登入或完成雙重驗證；本服務不收集平台憑證。
* 系統必須從官方 OAuth 身分回應確認正式帳號名稱，並讓使用者再次確認。
* 任何封鎖都必須綁定一個近期已重新載入的目標個人檔案。
* 封鎖前必須能寫入證據與稽核資料。

# 功能邊界

| 能力 | 是否納入 | 邊界 |
|---|---|---|
| 使用者註冊與登入 | 是 | 用於本服務的身分與租戶，不代表 Threads 授權。 |
| Threads 自助授權 | 是 | 憑證只由 Meta 官方 OAuth 頁面處理；服務只保存每連線加密 token。 |
| 候選變形產生 | 是 | 規則有限、可解釋、有每日與單次上限。 |
| 公開個人檔案檢查 | 是 | 只檢查已產生或人工加入的明確候選。 |
| 相似度排序 | 是 | 只用於排定審核順序。 |
| 自動封鎖 | 否 | 排程與相似度不得自行產生封鎖批准。 |
| 使用者批准後單一封鎖 | 是 | 一次只處理一個完整帳號名稱。 |
| 自動檢舉 | 否 | 檢舉原因與陳述需要使用者判斷，且平台流程可能要求額外資料。 |
| Meta 解除授權與資料刪除 | 是 | 只接受通過 `signed_request` HMAC、`issued_at` 格式與 future-skew 驗證的精確 callback；清理只限相符 Meta 身分及 cutoff 前資料。 |
| 刪除本服務帳號 | 尚未 | Meta 資料刪除不等同刪除 Cloudflare Access 使用者或整個應用租戶。 |
| CAPTCHA 或驗證繞過 | 否 | 一律暫停並交回使用者。 |
| 大規模帳號抓取 | 否 | 不建立通用搜尋器或全站列舉器。 |

# 成功條件

* 使用者不需要把 Threads 密碼、Cookie、Session 檔或雙重驗證碼傳給營運者。
* 使用者可以清楚看到目前連線的是哪個 Threads 帳號。
* 每個候選都有來源規則、相似理由、最新檢查時間與證據。
* 使用者能在一個頁面完成「忽略、持續監看或封鎖」決策。
* 發生不確定狀態時，系統不會重複或擴大破壞性動作。
* 使用者可立即撤銷連線並使雲端工作階段失效。
* Meta 解除授權與資料刪除重送不會擴大範圍或產生重複破壞性動作。

# 非功能目標

* 安全性優先於完全自動化。
* 多租戶隔離優先於共用瀏覽器效率。
* 可解釋候選優先於不可說明的黑箱分數。
* 最小資料收集優先於長期保存完整瀏覽紀錄。
* 低頻、可控的掃描優先於即時且高頻的輪詢。
